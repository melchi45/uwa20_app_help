Parking detection

Detects if the parking area you set is occupied and if it is, an event is triggered. It considers full when the number of vehicles parked in that area reaches the maximum number of parked vehicles you set.

Vehicles include automobiles, bicycles, and motor cycles.

Once the setup is complete, click the [Apply] button at the bottom of the page. This parking detection event setup also applies to [Parking detection] in [Event]>[Event setup] page.

Select channel

More parking detection details can be set for each channel after a camera channel is selected.

Channel 1 shows the whole feed from the camera. Channels 2 through 5 show the quad view of the whole feed. Channel 2 shows the top left section of the feed; Channel 3, the top right of the feed; Channel 4, the bottom left of the feed; and Channel 5, bottom right of the feed. The [Parking detection] menu can only be set in Channels 2 to 5.

Parking detection

Enable parking detection

To enable parking detection events, select [Enable parking detection].

Area

Sets the details for the parking detection area.

Parking area

Sets the ROI area where parked vehicles will be detected and the exclude area where parked vehicles won't be detected.

Also, the maximum number of vehicles needed to consider the parking lot as full can be set in [Max. vehicles]. In other words, the parking lot is considered full only when the number of parked vehicles inside the detection area reaches the number set in [Max. vehicles].

Setting ROI area and exclude area

exclude areas 1 and 2 are set in order after the ROI area is set. When either the ROI area or the Exclude area is configured, the color of the number buttons change.

  • The area can be set by selecting four points on the camera video screen or drag the cursor to draw a square.

Changing ROI area and exclude area

  • The area can be resized by moving the points of the specified area.
  • Point the line inside the area with the mouse cursor and a + mark appears. Click the + mark to add a point. A polygonal shape can be created by selecting up to 4 points.
  • To relocate the area, select the area and drag it to a desired location.

Deleting ROI area and exclude area

  • To delete an ROI area or an exclude area, right-click on the area to delete. Click the [OK] button from the Confirm Deletion window.

Common

Sets the sensitivity for parking detection and the minimum and maximum size of vehicles to detect. This setting is used to increase the detection accuracy and minimize false detection in the environment.

Sensitivity

Sets the sensitivity for parking detection. Set the sensitivity low in an environment where false detection of full status happens a lot, and set it high in an environment where false detection of vacant status happens a lot.

Size

Sets the minimum and maximum size of vehicles to detect.

Note

  • Vehicles smaller than the minimum size or bigger than the maximum size are not detected. To avoid false detection caused by noises, it's recommended to manually set the minimum and maximum size of vehicles to be detected in the actual installation environment. Please note that there could be some deviation in the detected size of a vehicle even if the same movement was made by the vehicle in the same place. Consider this when setting the minimum/maximum size restriction.
  • False detection may occur in the following conditions:
    • When the color or brightness of vehicle is similar to those of the background
    • When a number of motions are kept being detected randomly due to camera panning or other causes
    • When a fixed object constantly makes a movement on the spot
    • When random objects block different objects
    • When multiple objects merge into one or an object splits into multiple objects
    • When OSD menu of the camera is adjusted
    • When an object moves too fast
    • When reflection/blur/shadow occurs due to a strong light source such as direct sunlight, illumination, and head lights
    • When it rains or snows heavily, the wind blows hard, or it's dawn or dusk
    • If a moving object is approaching the camera
    • If it is not an indoor parking lot
    • If it is not a recommended installation height
    • If input video is unclear due to dirt or frost on lens
    • If a vehicle is parked sideways within the set area
    • If a vehicle is parked partially within the set area

Event action settings

E-mail

To send an image by e-mail when a parking detection event occurs, select [Enable]. Detailed settings for e-mail can be done from [Event]>[FTP/E-mail].

Alarm output

It is possible to set the alarm output number for a specific alarm when a parking detection event is created. After selecting an alarm output number, select [Off] if you wish no alarm, or select the output time if you wish to output an alarm. You can set a continuous alarm until turned off by the user by selecting [Always] or set the alarm for 5, 10 or 15 seconds.

Depending on the alarm output setup, [Alarm output] options may look different. If there is no alarm output, [Alarm output] options are not displayed. If there is more than one alarm output, they are displayed as [Alarm output 1], [Alarm output 2], and so on. Select [Event]>[Alarm I/O] to set the alarm I/O.

Goto preset

You can set the position of camera view to move to a preset number when a parking detection event is created. The camera preset can be set in [PTZ]>[Digital PTZ] or [Preset] of [External PTZ].

If [Enable external PTZ] option is deselected in [PTZ]>[External PTZ], [Goto preset] option is not displayed.

Audio clip

This enables playback of a specific audio clip when a parking detection event occurs. Once you select a file from your audio clip list, the clip will start to play when the event occurs. To disable the audio clip from playing when an event occurs, select [Off]. Clicking the Play button () allows you to play and preview an audio file.

Go to [Video & Audio]>[Audio setup]>[Audio clip] to register an audio clip.

Event activation time

This sets the operation time of the parking detection event action settings. Set the event action settings from [Event action settings] to always operate, or only at the set time.

Always

The event action settings are always executed when a parking detection event is created.

Only scheduled time

The event action settings are executed only when a parking detection event is created during the set time. The date from Sunday to Saturday, based on the current camera time, is displayed on the time table shown when [Only scheduled time] is selected. You can set the time for event action settings by clicking or dragging on the time table with the mouse. The set event activation time is executed repeatedly on the relevant weekday and time.

You can change the time view unit by clicking the [1 min], [30 min] or [1 h] button. When you click the [Reset] button, all set event activation times will be deleted. To confirm or change the camera time, refer to [Basic]>[Date & Time].
