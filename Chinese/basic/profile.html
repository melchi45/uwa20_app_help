视频配置文件

用户可以添加或删除视频配置文件，并更改配置文件属性。首先在“视频配置文件”中设置视频配置文件、帧速率以及编解码器，然后将视频配置文件改为串流图像或播放图像。完成设置后，单击页面底部的[应用]按钮。

选择 PTZ 模式

选中[外部 PTZ]时，用户可通过与 RS-485 或 RS-422 终端相连的接收器控制摄像机 PTZ。从 web 查看器左侧的菜单树中，可打开[PTZ] > [外部 PTZ]菜单。

选中[数字 PTZ]时，可对摄像机 PTZ 进行数字化控制。从 web 查看器左侧的菜单树中，可打开[PTZ] > [数字 PTZ]菜单。

视频配置文件连接策略

更改配置文件属性后，可以选择使用之前的配置文件或更改后的配置文件进行图像输出。

选中[在更改配置文件设置时保持连接]后，图像按照之前的配置文件属性输出，即使配置文件属性已经做出了更改；当重新连接网络查看器后，图像将按照更改后的配置文件进行输出。未选择[在更改配置文件设置时保持连接]时，配置文件属性更改后，图像将立刻按照更改后的配置文件进行输出。当配置文件已经更改后，在[正在监视]页面的实时视图边缘将变为橙色。

视频配置文件

用户可根据服务环境和产品使用场合选择视频配置文件。除默认的配置文件之外，用户还可以添加新的配置文件或删除已有的配置文件。您可以为每一个配置文件设置编解码器、配置文件类型、音频输入、ATC 模式、ATC 灵敏度、ATC 限制、分辨率、帧速率、最大比特率、目标比特率、比特率控制以及多播。

添加视频配置文件

  1. 单击[添加]按钮。将一个新项目添加至配置文件列表中。
  2. 在[名称]字段输入配置文件名称。所输入的名称将出现在配置文件列表中。
  3. 设置配置文件项目，包括[编解码器]、[配置文件类型]和[分辨率]。
  4. 单击页面底部的[应用]按钮。
  5. 弹出确认窗口时，单击[确认]按钮。新配置文件将被添加。

更改视频配置文件属性

  1. 从配置文件列表中选择想要更改的配置文件。
  2. 更改相应设置，包括[编解码器]、[配置文件类型]和[分辨率]。
  3. 单击页面底部的[应用]按钮。
  4. 弹出确认窗口时，单击[确认]按钮。所选的配置文件属性将被更改。

删除视频配置文件

  1. 从配置文件列表中选择一个想要删除的配置文件。
  2. 单击[删除]按钮。
  3. 弹出确认窗口时，单击[确认]按钮。所选定的配置文件将被删除。

配置文件类型

选择一个要应用的配置文件类型。选中的配置文件类型将被显示在配置文件列表的[类型]栏中。设置项目可能会因所选编解码器的类型而异。

  • 默认配置文件：这是应用于串流实时摄像机图像的默认配置文件。在配置文件列表的[类型]中显示为‘默认’。
  • 电子邮件/FTP 配置文件：创建一个事件时，此视频配置文件用于发送已捕获的屏幕图像。在配置文件列表的[类型]中显示为‘事件’。只有在编解码器设置为 MJPEG 时才会出现电子邮件/FTP 配置文件选项。
  • 录制配置文件：这种配置文件用于在 SD 卡或 NAS 上录制图像。在配置文件列表的[类型]中显示为‘录制’。
  • 数字 PTZ 配置文件：这种配置文件用于控制 PTZ 移动。在配置文件列表的[类型]中显示为‘DPTZ’。只有在编解码器设置为 H.264 或 H.265 时才会出现 数字 PTZ 配置文件选项。
  • 在鱼眼摄像机中添加新配置文件时，在[鱼眼视图类型]中选择[去翘曲视图]后，在[视图]中选择[四元视图]，自动将相应配置文件注册成数字PTZ 配置文件。

音频输入

当摄像机配备了内置麦克风或是连接有外部麦克风时，可以在图像中设置外部声音输入。

如果要使用监视页面中的麦克风和扬声器功能，应将[音频输入]设置为[启用]。

ATC 模式

ATC（自动传输控制）模式将根据网络带宽的变化更改图像的属性，从而调整发送的数据量。发送量调整的原理因 ATC 模式而异。

  • 禁用：即使网络带宽发生变化，固定的数据发送数量仍保持不变。
  • 启用 - 控制帧速率：如果网络带宽发生变化，可以通过更改帧速率来调整将要发送的数据量。
  • 启用 - 控制压缩：如果网络带宽发生变化，可以通过更改数据压缩率来调整将要发送的数据量。
  • 开 - 事件 (MD)：设置好移动探测事件后，可以使用此模式。当已经创建一个移动探测事件，且网络带宽发生变化时，可以通过更改帧速率来调整数据量。当没有探测到移动时，通过输出最少的帧数降低所占用的带宽。

灵敏度

调整网络带宽变化的反应速度。当 ATC 灵敏度非常高时，反应速度最快，当灵敏度非常低时，反应速度最慢。

只有在[ATC 模式]中选择了[启用 - 控制帧速率]或[启用 - 控制压缩]后，ATC 灵敏度才会被激活。

限制

根据网络带宽变化更改图像属性时，将 ATC 模式限制设置为更改属性所需的特定度。假设不使用 ATC 模式时为 100%，该值可设置为 10~50%，在此设置之下的属性将不会发生改变。

只有在[ATC 模式]中选择了[启用 - 控制帧速率]或[启用 - 控制压缩]后，ATC 限制才会被激活。

裁剪编码

裁剪编码这项功能可以选择并发送所需的特定区域；在每个配置文件中都可以选择、观看或流式传输不同的区域。如需在当前的配置文件中进行裁剪编码，请选择[启用]。

设置一个裁切区域

  1. 将裁剪编码设置为[启用]，然后单击[设置区域]按钮。
  2. 在[设置裁切区域]预览屏幕中拖动鼠标，设置区域。如需按 16:9 或 4:3 的比例设置区域，请选择[16:9]或[4:3]；如需按其它比例设置区域，请选择[手动]。
  3. 如需完成区域设置，请单击[确认]按钮。

帧速率

设置每秒的帧数。可用的帧速率范围会因[视频及音频] > [摄像机设置] > [传感器]中选择的帧速率而不同。

最大比特率

设置[比特率控制]为[VBR]时图像的最大比特率。如果待发送的图像数据大小没有超过设定的最大比特率，则将发送原图。

目标比特率

确定[比特率控制]为[CBR]时待发送的图像数据大小。

高级

如果将配置文件的[编解码器]设置为[H.264]或[H.265]，将显示全部的高级设置项。如果选择[MJPEG]作为配置文件的[编解码器]，将只显示[编码优先权]项。

比特率控制

设置调整图像数据大小的方式。

  • CBR：固定比特率 (CBR) 用于发送固定大小的全帧数据。选择 CBR 时，待发送的数据大小将由目标比特率的设置决定。CBR 具有固定的数据大小，因而能够轻松估计整个系统的数据大小，保证系统的稳定运行。
  • VBR：在保证用户所设置的视频属性的情况下，动态比特率 (VBR) 用于在确保最大比特率时发送图像，而不调整帧的数据大小。在兼顾质量的同时，VBR 可使用空余的存储空间或有效利用带宽，但如果图像突然变得复杂，则可能会给网络带来压力。

编码优先级

如图像数据大小超过目标比特率，则要设置帧速率和图像质量的优先权。

如果选择[H.264]或[H.265]作为配置文件的[编解码器]，用户可在[帧速率]和[压缩]之间进行选择。如果[帧速率]设置为高优先级，可保证最大帧速率，但图像质量会有所下降。而将[压缩]设置为高优先级时，可保证图像质量，但可能省略某些帧，可导致图像不连续或不自然。如果选择[H.264]或[H.265]作为配置文件的[编解码器]，仅当[比特率控制]设置为[CBR]时，[编码优先权]才能被激活。

如果配置文件的[编解码器]为[MJPEG]，可以在[帧速率]和[比特率]之间进行选

择。

视频组长度

GOV（视频组）是一组用于 H.264/H.265 视频压缩的图像帧，表示一个图像帧与下一个图像帧之间的帧组。GOV 包含图像帧和 P 帧。图像帧是压缩的基础（也被称为关键帧）；包含一张完整图像的数据。P 帧只包含在图像帧的基础上已更改过的区域的信息。因此，视频组越长，图像帧的数量越少，图像越小，而视频组越短，图像帧的数量越多，图像越大。视频组长度的最大值因[配置文件属性]下的[帧速率]而不同。

配置文件

只有当配置文件的编解码器为 H.264 时，此菜单才会被激活。配置文件可被视为一系列不同压缩技术的集合。Hanwha Techwin 摄像机所支持的配置文件包括[基础]、[主要]以及[高]。从基础转移至高时，压缩效果更好，质量更佳，但是压缩和解压会占用大量的系统资源，可能会对设备播放造成压力。

熵编码

设置编码类型，降低压缩损失。有两种熵编码类型可供选择：CAVLC（语境自适应长度可变编码）以及 CABAC（语境自适应二进制编码）。

  • CABAC：CABAC 的数据处理流程比 CAVLC 更加复杂，占用的系统资源更多，但具有绝佳的压缩率。
  • CAVLC：CAVLC 的数据处理流程比 CABAC 简单，占用的系统资源更少，但压缩率相对较低。

智能编码

设置是否使用智能编码。智能编码是 Hanwha Techwin 公司的独有技术，可以针对用户感兴趣的区域降低压缩率（提高输出质量），同时增加其他区域的压缩率，（输出正常质量），从而降低整体图像的数据大小。只有当[比特率控制]为[CBR]时，智能编码才会被激活。

智能编码的区域可以通过[视频及音频] > [智能编码]进行设置。

动态 GOV

如需在当前配置文件中应用动态 GOV 功能，请选择[启用]。动态 GOV 是一种视频组长度随图像状况进行自动变化的功能。换而言之，如果图像几乎没有移动，用户在[动态 GOV]中设置的视频组长度将增加到最大值，而检测到移动时，它的长度将降回[视频组长度]中设置的长度，以输出图像。只有当[比特率控制]为[VBR]时，动态 GOV 才会被激活。

  • 请输入图像没有移动时所使用的最大视频组长度。输入值的范围显示在[视频组长度]旁。当最大值为 480 时，[视频组长度]的输入值变为最小值，并根据[帧速率]中的[配置文件属性]而改变。

须知

  • 使用WiseStream功能时， 如果使用动态 GOV和动态 FPS功能， 可使WiseStream性能得到优化。WiseStream可在[视频及音频] > [WiseStream]菜单中进行设置。

动态 FPS

如果要将动态FPS功能应用到目前正在设置的配置文件中，则选择[启用]动态FPS是根据视频情况自动修改每秒帧数(frames per second)的功能。动态FPS只有在将[编解码器]选为[H.264]或[H.265]，[比特率控制]为[VBR]时才可激活。

最小 FPS

请输入动态 FPS运行时应用的最小FPS值。

须知

  • 使用WiseStream功能时， 如果使用动态 GOV和动态 FPS功能， 可使WiseStream性能得到优化。WiseStream可在[视频及音频] > [WiseStream]菜单中进行设置。

多播

多播是一种将一个实例的数据从摄像机发送至多个设备的方法。设置是否在当前配置文件中使用 RTSP（实时串流协议），并输入具体信息。

多播（RTSP）

如需要使用 RTSP 发送图像，请选择[启用]。

IP 地址

输入可以与 IPv4 网络连接的 IPv4 地址。如果多播地址设置为 224.0.0.0 ~ 224.0.0.255，那么在特定环境下无法使用多播。如果出现问题，请更改多播地址。

端口

设置控制发送图像的端口。多播 RTSP 的端口范围为 1024 到 65535。(但是不能使用 3702 端口。）

TTL

可以设置 RTSP 数据包的 TTL。可以输入 0 到 255 的 TTL 值。
