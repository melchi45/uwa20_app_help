SNMP

SNMP（简单网络管理协议）是一种可以从网络上的设备收集信息并管理网络的网络管理协议。完成设置后，单击页面底部的[应用]按钮。

SNMP v1/v2c

SNMP v1 协议未加密，且几乎没有安全功能。此外，因为占用的宽带较多，如果网络中存在较多设备，则很难对网络进行管理。SNMP v2c 协议为确保数据安全和验证安全，加入了一种运算法，在 SNMP v1 基础上增加了宽带利用率。

SNMP v1

如需使用 SNMP v1，请选择[启用]。

SNMP v2c

如需使用 SNMP v2c，请选择[启用]。选择[SNMP v2c]时，将启用读取社区和写入社区。

读取社区

输入只读社区的名称，访问 SNMP 信息。默认值为“公共”。

写入社区

输入只写社区的名称，访问 SNMP 信息。默认值为“写入”。

SNMP v3

（仅在 SSL/TLS 经过验证后操作）。

SNMP v3 验证比 v1 和 v2c 更安全，无需数据转换即可进行传输。此外，数据包进行了加密，防止未授权用户访问数据。只有在设置了 SSL/TLS 的情况下才能使用 SNMP v3。

SNMP v3

如需使用 SNMP v3，请选择[启用]。

密码

为 SNMP v3 设置用户密码。密码必须为 8-16 个字符。默认密码的安全性较低，因此强烈建议在安装产品后，立即更改密码。重要提示：用户自行负责密码的安全以及因继续使用默认密码而引发的其它问题。

说明

  • 仅当安全连接系统为 HTTPS 模式时，才可以设置 SNMP v3。
  • 不使用 SNMP v3 可导致安全问题。

SNMP 陷阱

SNMP 陷阱功能可将网络中设备的具体事件发送给管理系统。

SNMP 陷阱

如需使用 SNMP 陷阱，请选择[启用]。

社区

输入接收消息的陷阱社区的名称。

IP 地址

输入消息发送用户的 IP 地址。

  • 认证失败：设置是否当社区信息不正确时向管理系统发送一个事件。
  • 网络连接：设置是否重新连接已断开网络时向管理系统发送一个事件。
